JCP - Java Community Process

  • Formalized mechanism for developing Java specifications
  • Not standards organization / legal entity - dominated by Oracle
  • JSR - Java Specification Request:
    • Specification, Reference Implementation & TCK
    • TCK - Technology Compatibility Kit
  • Criticized as "rubber stamp organization", but:
    • Apache vs. Sun on getting access to the TCK
    • JCP EC rejects JSR 376 (Jigsaw)
  • Three Java platforms: Java ME, Java SE and Java EE ⇒EE4J

OpenJDK

  • Java SE reference implementation (since Java 7)
  • GPLv2 plus Classpath Exception
  • "Source-only" project (until Java 8)
    • Since Java 9 binaries for Linux/x64, MacOS, Windows
      ⇒ Two updates only!
  • Binary distributions from various vendors

OpenJDK Updates Projects

  • JDK 6 ⇒ maintained by Azul
  • JDK 7 Updates ⇒ maintained by Red Hat
  • JDK 8 Updates ⇒ maintained by Red Hat
  • JDK 9 Updates ⇒ abandoned after two releases
  • JDK 10 Updates ⇒ abandoned after two releases
  • JDK 11 Updates ⇒ ???

Oracle JDK

  • Derived from Open JDK
    • with non - Java SE addons (e.g. JavaFX, WebStart)
    • and commercial features (e.g. Flight Recorder, Mission Control)
  • Under Oracle Binary Code License (until 8u202)
    • Allows commercial use but
      ⇒ Field of Use restrictions may apply
  • Under Oracle Technology Network License (after 8u202)
    • Only for personal / development use!

JCK - Java Compatibility Kit

  • TCK for Java SE conformance certification
  • Available only to commercial Java licensees or
  • OpenJDK Community TCK License Agreement (OCTLA) signatories
    • Only for GPL licensed implementations
      substantially derived from the OpenJDK
    • Closed Source; OCTLA ⇒ NDA agreement
  • Complex test suite (~140_000 tests, JCK 6 Users Guide ~300p)

Java Versioning

  • Good old days: 6, 7, 8, but Updates Version scheme complex:
    • Limited update releases, Critical Patch Updates, Patch Set Update
      8u20, 8u25, 8u31, 8u40, 8u45, 8u51, 8u60, 8u65, 8u66
  • JEP 223: New Version-String Scheme
    • 9, 9.0.1, 9.0.4
  • JEP 322: Time-Based Release Versioning
    • Introduced java.vendor.version & the notion of LTS
    • 10 (18.3), 10.0.1 (18.3), 10.0.2 (18.3)
    • 11 LTS (18.9), 11.0.1 LTS (18.9), 10.0.2 LTS (18.9)
  • 8216383: Remove the system property "java.vendor.version"
